**(잡생각) 제약과 목표**

### state 와 transition 으로 정의된 system
- 제약: initial states
- 목표: goal state
- 문제 해결: inital states 에서 시작하여 goal state 에 도달하는 것.
- 목표는 system 을 탐험하는 agent 가 설정하는 것.

### 제약과 목표 유무에 따른 분류
- 제약 X, 목표 X: 아무런 state 가 존재하지 않는 공허한 시스템
- 제약 X, 목표 O: 공허한 시스템안에서 agent 가 goal state 를 마음껏 만들고 이에 곧바로 도달할수 있는 시스템
- 제약 O, 목표 X: 법률/행정 시스템 및 수학/과학 이론, 게임 엔진
- 제약 O, 목표 O: 수학/코딩 및 각종 공학/이학적 최적화 문제, 게임 플레이

### 문제 해결을 위해서는 시스템의 제약과 목표를 명확히하고 어떤 룰을 가지고 있는지 중요함.
- 시스템의 분절이 가능한 경우, 어떤 시스템인지 특정하고 해당 시스템의 룰을 따라야함.
   - real world 와 video game 의 물리 엔진은 완전히 같지 않음.
   - video game 의 물리 엔진이 기동되기 위해서 real world 의 전기와 실리콘이 필요하지만, 룰 자체는 독립적으로 존재함.
   - A 사의 내규인지 B 사의 내규인지, 법률 문제인지, 사회적 규범에 어긋나는지 등 구분이 필요.

### 시스템 A 에 존재하는 state 와 transition rule 이 시스템 B 에서는 전혀 다른 state, transition rule 에 대응될수 있음
- state: 모델링 하고자 하는 대상이 '시스템 내에서 어떻게 레이블되느냐' 로 대상 그 자체가 아님.
- transition rule 이 전혀 다를수도 있음.
- 서로 다른 사람이 같은 물체/현상을 경험하고 다르게 느끼는 것은 당연함.
